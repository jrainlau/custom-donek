# 需求文档

## 引言

本项目是一个 **Donek 滑雪板图案颜色定制 Web 应用**，基于 Vite + React + TypeScript 从零搭建，纯前端项目，不涉及后端。

用户可以在页面上分别对滑雪板的**板面（Topsheet）**和**板底（Base）**进行颜色定制。每一面各有 2 个可配置的颜色（共 4 个颜色），对应 SVG 中的两类填充色。颜色修改后在页面上实时预览。应用还提供预设配色模板、用户自定义配色保存（IndexedDB），以及一键导出包含配色信息和板面/板底预览的效果图 PNG。

参考图展示了一种卡片式布局：上方为板面小图+板底大图的展示区，右侧为配色色块与颜色值，整体风格简洁现代，以浅米色/奶油色为背景。本项目将参考但不完全照搬该布局。

### 颜色编码规范

- 内部数据存储和处理可以使用 HEX 编码
- **所有面向用户的颜色展示必须使用 RGB 格式**（如 `RGB(211, 73, 55)`）
- 每个颜色色块/选择器下方都必须标明对应的 RGB 值
- 用户可以通过点击 RGB 值一键复制到剪贴板

### SVG 资源说明（已优化）

原始 SVG 存放于 `assets/` 目录。由于原始 SVG 中板面包含十多种不同的灰度色值，板底缺少背景矩形，不便于运行时颜色替换，因此已生成优化版本存放于 `src/assets/` 目录：

- **src/assets/Topsheet.svg（板面）**：所有路径已通过 CSS class 标记为两类颜色：
  - `.color-primary`：大面积主体色（原 `#000000`，默认黑色）
  - `.color-secondary`：文字/细节色（原多种灰度值统一归为 `#F5F5F5`，默认浅灰）
- **src/assets/Base.svg（板底）**：已添加底层背景矩形，通过 CSS class 标记为两类颜色：
  - `.color-bg`：背景色（默认 `#FFFFFF` 白色）
  - `.color-pattern`：图案色（默认 `#000000` 黑色）

### 颜色替换策略

运行时通过 JS 查询 SVG DOM 中的 CSS class，修改对应 `<style>` 标签中的 fill 规则来实现颜色替换：

- **板面**：修改 `.color-primary` 的 fill 为用户选择的「板面颜色 1」；修改 `.color-secondary` 的 fill 为用户选择的「板面颜色 2」
- **板底**：修改 `.color-pattern` 的 fill 为用户选择的「板底颜色 1」；修改 `.color-bg` 的 fill 为用户选择的「板底颜色 2」

## 需求

### 需求 1：项目初始化与基础架构

**用户故事：** 作为一名开发者，我希望项目基于 Vite + React + TypeScript 搭建，以便拥有现代化的开发体验和类型安全

#### 验收标准

1. WHEN 开发者克隆项目并执行 `npm install && npm run dev` THEN 系统 SHALL 成功启动开发服务器并在浏览器中展示页面
2. IF 项目使用 TypeScript THEN 系统 SHALL 对所有组件和工具函数进行类型定义，确保无 TypeScript 编译错误
3. WHEN 项目构建 THEN 系统 SHALL 输出可部署的静态资源文件

---

### 需求 2：SVG 颜色实时替换与预览

**用户故事：** 作为一名用户，我希望能够为滑雪板的板面和板底分别选择 2 个颜色，并在页面上实时看到 SVG 图案的颜色变化，以便直观地设计我喜欢的配色方案

#### 验收标准

1. WHEN 页面加载完成 THEN 系统 SHALL 展示板面（Topsheet）和板底（Base）的 SVG 预览，使用默认颜色渲染
2. WHEN 用户调整「板面颜色 1」THEN 系统 SHALL 实时修改板面 SVG 中 `.color-primary` class 对应的 fill 颜色
3. WHEN 用户调整「板面颜色 2」THEN 系统 SHALL 实时修改板面 SVG 中 `.color-secondary` class 对应的 fill 颜色
4. WHEN 用户调整「板底颜色 1」THEN 系统 SHALL 实时修改板底 SVG 中 `.color-pattern` class 对应的 fill 颜色
5. WHEN 用户调整「板底颜色 2」THEN 系统 SHALL 实时修改板底 SVG 中 `.color-bg` class（背景矩形）对应的 fill 颜色
6. WHEN 用户调整任意颜色 THEN 系统 SHALL 在 100ms 以内完成 SVG 的重新渲染，确保视觉上无明显延迟

---

### 需求 3：RGB 颜色选择器

**用户故事：** 作为一名用户，我希望通过 RGB 颜色选择器来选取颜色，以便精确控制配色

#### 验收标准

1. WHEN 用户点击颜色区域 THEN 系统 SHALL 展示一个 RGB 颜色选择器（不含透明度/Alpha 通道）
2. IF 颜色选择器使用开源库 THEN 系统 SHALL 选用成熟且轻量的开源 React 颜色选择器组件（如 `react-colorful`）
3. WHEN 用户在颜色选择器中拖动选色区域或输入 RGB 值 THEN 系统 SHALL 实时更新对应的 SVG 预览颜色
4. WHEN 用户选择颜色 THEN 系统 SHALL 在颜色色块下方展示该颜色的 RGB 值（如 `RGB(211, 73, 55)`），方便用户记录
5. WHEN 用户点击 RGB 值文本 THEN 系统 SHALL 将该 RGB 值复制到系统剪贴板，并展示「已复制」的短暂提示

---

### 需求 4：预设配色模板

**用户故事：** 作为一名用户，我希望能够从预设的配色模板中快速选择，以便在没有设计灵感时也能获得好看的配色方案

#### 验收标准

1. WHEN 页面加载完成 THEN 系统 SHALL 展示 10 套不同风格的预设配色模板，每套包含 4 个 RGB 颜色（板面颜色 1、板面颜色 2、板底颜色 1、板底颜色 2）
2. WHEN 用户点击某套配色模板 THEN 系统 SHALL 立即将 4 个颜色应用到对应的 SVG 预览上
3. IF 配色模板被选中 THEN 系统 SHALL 高亮显示当前选中的模板
4. WHEN 展示配色模板 THEN 系统 SHALL 为每套模板标注一个风格名称（如"苹果肉桂"、"极地冰蓝"等），并以 4 个色圆形式展示颜色

### 需求 5：用户自定义配色保存

**用户故事：** 作为一名用户，我希望能够保存自己调配好的配色方案到本地，以便下次访问时可以快速恢复和使用

#### 验收标准

1. WHEN 用户点击"保存配色"按钮 THEN 系统 SHALL 弹出一个输入框让用户为配色方案命名
2. WHEN 用户确认保存 THEN 系统 SHALL 将当前 4 个颜色值及名称存储到浏览器的 IndexedDB 中
3. WHEN 页面加载完成 THEN 系统 SHALL 从 IndexedDB 中读取已保存的配色方案并展示在"我的配色"区域
4. WHEN 用户点击已保存的配色方案 THEN 系统 SHALL 立即将该配色应用到 SVG 预览上
5. WHEN 用户希望删除已保存的配色 THEN 系统 SHALL 提供删除功能，并从 IndexedDB 中移除对应记录
6. IF IndexedDB 不可用 THEN 系统 SHALL 降级使用 localStorage 作为存储方案，并提示用户存储容量有限

---

### 需求 6：一键导出效果图 PNG

**用户故事：** 作为一名用户，我希望能够一键导出包含板面、板底和配色信息的效果图 PNG，以便分享给朋友或提交定制订单

#### 验收标准

1. WHEN 用户点击"导出效果图"按钮 THEN 系统 SHALL 生成一张 PNG 图片并触发浏览器下载
2. WHEN 导出效果图 THEN 系统 SHALL 在图片中同时包含：
   - 板面 SVG 的当前配色预览
   - 板底 SVG 的当前配色预览
   - 4 个颜色色块及其 RGB 值标注
3. WHEN 导出效果图 THEN 系统 SHALL 使用 Canvas API 或 `html-to-image` 等库将 DOM 渲染为 PNG
4. IF 导出过程耗时较长 THEN 系统 SHALL 展示加载状态提示，避免用户重复点击
5. WHEN 导出完成 THEN 系统 SHALL 生成分辨率不低于 1920×1080 的高清图片

### 需求 7：页面 UI 设计与交互

**用户故事：** 作为一名用户，我希望页面美观、现代、易用，以便获得良好的配色设计体验

#### 验收标准

1. WHEN 页面加载 THEN 系统 SHALL 展示简洁现代的布局，采用浅暖色调（如奶油色/米色）背景，参考但不照搬参考图风格
2. WHEN 页面展示 THEN 系统 SHALL 包含以下区域：
   - **预览区**：左侧或中央，展示板面和板底的 SVG 预览（板面为小尺寸带标签的预览 + 板底为大尺寸详细预览，或并排展示）
   - **颜色控制区**：右侧或下方，包含 4 个颜色选择器（板面颜色 1/2，板底颜色 1/2）
   - **配色模板区**：下方或侧边，展示预设模板和用户保存的配色方案
   - **操作区**：包含"保存配色"和"导出效果图"按钮
3. WHEN 用户在不同屏幕尺寸下访问 THEN 系统 SHALL 提供基本的响应式布局，确保在桌面端（≥1024px）有良好的展示效果
4. WHEN 鼠标悬停在可交互元素上 THEN 系统 SHALL 展示适当的 hover 效果和光标变化
5. WHEN 用户进行颜色切换或模板切换 THEN 系统 SHALL 提供平滑的过渡动画效果

## 边界情况与注意事项

1. **SVG 已预优化**：优化版 SVG 已将所有颜色归为 CSS class，无需在运行时逐个替换多种灰度值。项目应使用 `src/assets/` 下的优化版 SVG，而非 `assets/` 下的原始文件
2. **内联 SVG 渲染**：SVG 需以内联方式嵌入 DOM（而非 `<img>` 标签），以便通过 JS 操作其内部 `<style>` 标签进行颜色替换
3. **颜色选择器无 Alpha 通道**：必须严格使用 RGB 模式，不能出现 RGBA 或透明度控制
4. **颜色展示格式**：内部处理可用 HEX，但所有面向用户的界面上必须显示 RGB 格式（如 `RGB(211, 73, 55)`），且每个颜色值支持点击复制
4. **导出图片质量**：Canvas 渲染 SVG 时可能出现模糊，需确保高 DPI 渲染
5. **IndexedDB 兼容性**：极少数浏览器或隐私模式下 IndexedDB 不可用，需提供 localStorage 降级方案
6. **性能**：颜色实时替换操作频繁（拖动选色器时），需注意防抖或优化渲染性能
