# 需求文档

## 引言

本次迭代为已有的 Donek 滑雪板配色定制应用新增 **Material Design 3 动态配色生成** 能力，并进行 **UI 布局优化**。

### 背景

Google 的 Material Design 3 提供了一套颜色算法（参考文档 https://m3.material.io/styles/color/dynamic/user-generated-source），允许用户选定一个"种子颜色"后自动生成一系列和谐的配套色。Google 官方提供了 npm 包 `@material/material-color-utilities`（GitHub 地址：https://github.com/nickytonline/material-color-utilities ），这是一个纯 TypeScript 库，可直接在前端使用，无需后端支持。

本次迭代将该算法集成到项目中，新增一个"智能配色"UI 模块。用户选定一个种子颜色后，系统自动从 M3 色彩方案中提取 4 个和谐颜色，分别赋予板面和板底的文字色与背景色。生成的 4 个颜色可以通过拖拽自由交换位置，方便用户尝试不同的搭配。

同时，本次迭代还将修复当前 UI 的两个问题：
1. **整体布局**：调整为左侧预览区 + 右侧功能区的明确分栏布局
2. **SVG 预览被遮挡**：SVG 预览应自适应容器宽高，确保完整显示

### 技术选型补充

| 技术项 | 选型 | 说明 |
|-------|------|------|
| M3 配色算法 | `@material/material-color-utilities` | Google 官方 npm 包，纯 TS 实现，支持从种子颜色生成完整的 Material 色彩方案 |
| 拖拽交互 | HTML5 Drag & Drop API 或轻量拖拽实现 | 用于 4 个颜色块之间的位置交换，无需引入额外拖拽库 |

## 需求

### 需求 1：集成 Material Design 3 动态配色生成

**用户故事：** 作为一名用户，我希望选定一个种子颜色后系统能自动生成 4 个和谐的配色，以便快速获得专业的色彩搭配而无需手动逐个调色

#### 验收标准

1. WHEN 用户在"智能配色"模块中选择一个种子颜色 THEN 系统 SHALL 使用 `@material/material-color-utilities` 的算法从该种子颜色生成一个完整的 M3 色彩方案
2. WHEN M3 色彩方案生成后 THEN 系统 SHALL 从方案中提取 4 个具有代表性的颜色（如 primary、onPrimary、primaryContainer、onPrimaryContainer 或其他合适的色调组合），分别赋予板面主体色、板面细节色、板底图案色、板底背景色
3. WHEN 智能配色应用后 THEN 系统 SHALL 实时更新 SVG 预览，展示新的配色效果
4. WHEN 用户调整种子颜色 THEN 系统 SHALL 在 200ms 内完成新色彩方案的生成与预览更新
5. IF `@material/material-color-utilities` 包不可用或加载失败 THEN 系统 SHALL 优雅降级，保留现有的手动选色功能不受影响

---

### 需求 2：4 色位置可拖拽交换

**用户故事：** 作为一名用户，我希望能够通过拖拽来自由交换 4 个颜色的应用位置，以便快速尝试不同的颜色分配方案而无需逐个重新选色

#### 验收标准

1. WHEN 智能配色生成 4 个颜色后 THEN 系统 SHALL 以 4 个可拖拽的色块形式展示，每个色块标注当前的应用位置（板面主体色、板面细节色、板底图案色、板底背景色）
2. WHEN 用户将一个色块拖拽到另一个色块的位置上 THEN 系统 SHALL 交换这两个颜色的应用位置，并实时更新 SVG 预览
3. WHEN 拖拽交互发生时 THEN 系统 SHALL 提供清晰的视觉反馈：
   - 被拖拽的色块应有提升/阴影效果
   - 拖拽经过的目标位置应有高亮提示
   - 交换完成后应有平滑的过渡动画
4. WHEN 用户完成拖拽交换后 THEN 系统 SHALL 在色块下方更新位置标签和 RGB 值
5. IF 设备不支持拖拽（如触控设备） THEN 系统 SHALL 提供备选的交互方式（如点击选中两个色块进行交换）

---

### 需求 3：UI 布局调整为左右分栏

**用户故事：** 作为一名用户，我希望页面左侧是预览区、右侧是所有功能区，以便在调整配色时能始终看到预览效果

#### 验收标准

1. WHEN 页面加载 THEN 系统 SHALL 采用左右分栏布局：
   - **左侧**：SVG 预览区（板面 + 板底的预览）
   - **右侧**：所有功能区域（颜色配置面板、智能配色模块、预设模板、用户自定义配色、导出按钮等）
2. WHEN 用户在右侧功能区进行操作（如选色、切换模板等） THEN 系统 SHALL 确保左侧预览区始终可见（预览区使用 sticky 定位或固定定位）
3. WHEN 右侧功能区内容较多需要滚动时 THEN 系统 SHALL 仅右侧区域可滚动，左侧预览区保持固定
4. WHEN 页面宽度 ≥ 1024px THEN 系统 SHALL 展示左右分栏布局，预览区与功能区的宽度比例约为 1:1 或适当比例
5. IF 页面宽度 < 1024px THEN 系统 SHALL 可以降级为上下布局（预览在上，功能区在下），不做强制要求

---

### 需求 4：SVG 预览自适应修复

**用户故事：** 作为一名用户，我希望 SVG 预览能完整显示在预览区域内而不被遮挡或截断，以便准确查看配色效果

#### 验收标准

1. WHEN SVG 预览渲染时 THEN 系统 SHALL 使 SVG 图形自适应预览容器的宽高，不超出容器边界
2. WHEN 预览容器尺寸变化（如窗口调整大小） THEN 系统 SHALL 自动调整 SVG 的显示尺寸以适配新的容器大小
3. WHEN SVG 自适应显示时 THEN 系统 SHALL 保持 SVG 原始宽高比例，不产生拉伸变形
4. WHEN SVG 预览区域展示板面和板底两个 SVG 时 THEN 系统 SHALL 合理分配空间，确保两个 SVG 都能完整显示
5. IF SVG 内容过长（滑雪板纵横比较大） THEN 系统 SHALL 使用 `object-fit: contain` 等策略确保 SVG 在容器内完整可见

---

## 边界情况与注意事项

1. **M3 库兼容性**：`@material/material-color-utilities` 是纯 TS/JS 库，在 Vite 环境下应可直接导入使用，但需注意版本兼容性和 Tree-shaking
2. **色彩方案提取策略**：M3 生成的色彩方案包含数十种色调，需要精心选择 4 个最适合滑雪板配色的色调。建议从 `primary`、`onPrimary`、`primaryContainer`、`onPrimaryContainer` 中选取，或允许用户在"light/dark"方案间切换
3. **拖拽性能**：拖拽过程中实时更新 SVG 预览可能有性能问题，可考虑在拖拽结束（drop）时才更新预览
4. **智能配色与手动配色的关系**：智能配色模块生成的颜色应可以被手动颜色选择器进一步微调，两者互不排斥
5. **已有功能不受影响**：本次迭代不应破坏已有的预设模板、用户自定义保存、导出 PNG 等功能
6. **SVG 自适应与导出的关系**：预览区的自适应是纯 CSS 表现层的调整，不应影响导出 PNG 时的 SVG 尺寸和质量
